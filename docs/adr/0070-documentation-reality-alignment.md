# 0070 - Documentation Reality Alignment Process

## Status

Accepted

## Context

README claims are user-facing promises.

## Decision

README feature claims must be either:

- Backed by a test or smoke assertion.
- Clearly marked as deployment/setup limitation.
- Removed.

The feature claims audit is updated when claims change.

## Consequences

Docs become a tested surface, not a wishlist.

## Alternatives Considered

Leaving aspirational claims in README was rejected because it wastes user time.
